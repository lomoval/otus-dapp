# HW1 - Подготовка и хранение простого смарт-контракта

## Шаги запуска проекта и тестов,

1. Установил NodeJS.
2. Установил пакеты при помощи скрипта: `./scripts/install-all.sh`.
3. Добавил 3 контракта: одно значение, массив, мапа.
3. Написал тесты.
4. Запустил проверку тестов: `npx hardhat test`.
5. Запуск локальной сети: `npx hardhat nod`.
6. Правка деплоя под 3 контракта.
7. Запуск деплоя: `npx hardhat run scripts/deploy.ts --network localhost`.
8. Заменил провайдера в `nextjs-frontend/components/EthereumWallet.tsx`  на локального - `const provider = new ethers.JsonRpcProvider("http://localhost:8545")`, чтобы можно было взять/установить значение у контракта с одним значением.
9. Запустил фронт в `nextjs-frontend`: `npm run dev`
10. Получил значение из деплоя и обновил его.

### Бонусная часть: Массивы vs Mapping

В специфику как их лучше использовать в контрактах не погружался, но базовые вещи должны быть, как и для других подходов:
- массивы: хорошо подходят для хранения упорядоченных данных, где важен порядок элементов.
- mapping: идеальны для хранения данных по ключу, обеспечивают быстрый доступ и подходят для динамичных данных.
